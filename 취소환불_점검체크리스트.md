# 취소·환불 결제취소 점검 체크리스트

실결제 오픈 전/후, 포트원 결제취소 + 쿠폰·포인트 복원이 제대로 도는지 확인하는 체크리스트.

---

## 0. 사전 설정 (실결제 오픈 시 1회)

- [ ] **`PORTONE_API_SECRET`** 이 Vercel 환경변수(Production)에 설정돼 있다
- [ ] 그 키가 **라이브(실결제) 키** 다 (테스트 키 아님) — 오픈 후
- [ ] 포트원 콘솔에서 해당 채널에 **결제취소(환불) 권한**이 켜져 있다
- [ ] 아래 SQL이 Supabase에서 실행돼 있다
  - [ ] `add_refund_type.sql` (refund_requests.type)
  - [ ] `add_refund_restore.sql` (orders.used_coupon_id / earned_point / refund_restored)
- [ ] 정산 계좌/환불 가능 잔액 등 PG사 환불 조건 확인 (PG마다 환불은 매입 후 가능 시점 다름)

---

## 1. 테스트 주문 만들기 (검증용)

각 케이스별로 주문을 1건씩 만들어두면 좋다.

- [ ] **A. 카드결제 + 쿠폰 + 포인트 사용** 주문 (가장 중요 — 복원 전부 확인)
- [ ] **B. 카드결제만** 주문 (쿠폰·포인트 없음)
- [ ] **C. 0원 주문** (쿠폰/포인트로 전액 차감 → 결제 ID 없음)

> ⚠️ 쿠폰·포인트 복원은 **이 배포 이후 새로 만든 주문**부터 동작한다(주문 시점에 used_coupon_id·earned_point를 저장). 기존 주문은 복원 데이터가 없다.

---

## 2. 경로 ①: 고객 신청 → 어드민 승인

### 주문취소 (배송 전)
- [ ] 마이페이지 주문내역에서 배송 전 주문에 **「주문취소」** 버튼이 보인다
- [ ] 사유 선택 후 신청 → "주문취소 신청 접수" + 버튼이 "취소 신청 접수" 뱃지로 바뀐다
- [ ] 어드민 **환불 관리**에 **유형: 취소신청** 으로 뜬다 (유형 필터 "취소"로 조회됨)
- [ ] 상세 → **승인(완료)** 클릭
- [ ] → **포트원에서 실제 결제취소됨** (아래 4번 검증)
- [ ] → 주문상태가 **취소됨(cancelled)** 으로 바뀐다
- [ ] → "복원 완료: 포인트 N P 환급 · 적립 M P 회수 · 쿠폰 복원" 알림이 뜬다

### 환불 (배송완료 후)
- [ ] 배송완료 주문에 **「환불신청」** 버튼이 보인다
- [ ] 신청 → 어드민에 **유형: 환불신청** 으로 뜬다
- [ ] 승인(완료) → 포트원 취소 + 주문상태 **환불완료(refunded)** + 복원 알림

### 거절
- [ ] 거절 + 사유 입력 → 마이페이지에 **"취소/환불 불가 사유"** 가 보인다
- [ ] 거절은 **포트원 호출 안 함**, 쿠폰·포인트 복원 **안 함** (정상)

---

## 3. 경로 ②: 어드민 직접 주문취소

- [ ] 어드민 주문관리 → 주문 상세 → **주문취소** 버튼
- [ ] → 포트원 결제취소 먼저 수행 후 상태 cancelled
- [ ] 카드취소 실패 시 **"카드 취소 실패 — 주문취소 중단"** 뜨고 상태 안 바뀐다 (정상)

> 참고: 직접취소는 신청 레코드가 없어 KPI 숫자엔 안 잡히고, 환불관리 목록엔 **관리자취소/관리자환불** 로 표시됨.

---

## 4. 포트원 실제 취소 검증

- [ ] **포트원 콘솔 → 결제내역**에서 해당 결제가 **취소/부분취소 상태**로 바뀌었다
- [ ] 취소 금액이 결제 금액과 일치한다 (현재 **전액취소만** 지원)
- [ ] 카드 매출전표/고객 카드사 기준 환불 처리됨 (영업일 소요될 수 있음)
- [ ] **이미 취소된 결제를 또 승인**해도 오류 없이 넘어간다 (멱등 — PAYMENT_ALREADY_CANCELLED 성공 처리)

---

## 5. 쿠폰·포인트 복원 검증 (케이스 A)

- [ ] 고객 **보유 포인트**가 (사용분 환급 − 적립분 회수) 만큼 정확히 조정됐다
- [ ] 고객 **마이페이지 → 포인트 내역**에 "사용 포인트 환급", "적립 포인트 회수" 기록이 남았다
- [ ] 사용했던 **쿠폰이 다시 미사용(보유)** 상태로 돌아왔다 (마이페이지 쿠폰함)
- [ ] **두 번 승인해도** 포인트/쿠폰이 **중복 복원되지 않는다** (orders.refund_restored 멱등)

---

## 6. 엣지 케이스

- [ ] **결제 ID 없는 0원 주문(C)** 승인 → "결제 ID 없어 상태만 변경" confirm 후, 포트원 호출 없이 상태만 바뀌고 쿠폰·포인트는 복원된다
- [ ] **`PORTONE_API_SECRET` 미설정** 상태에서 승인 시 → 503 / "포트원 API 시크릿 미설정" 뜨고 상태 안 바뀐다
- [ ] 같은 주문에 대해 **취소 신청과 환불 신청이 중복 접수되지 않는다** (진행 중이면 버튼 대신 뱃지)

---

## 미지원 / 향후 과제 (알고만 있기)

- 부분취소(부분환불) — 현재 전액만. 필요 시 `/api/payment/cancel` 에 `amount` 전달로 확장 가능
- 기존(배포 이전) 주문은 쿠폰·used_coupon_id/earned_point 스냅샷이 없어 자동 복원 불가 → 필요 시 수동 처리
- 무통장/가상계좌 등 비카드 결제의 환불 흐름은 PG 설정에 따라 별도 확인 필요
