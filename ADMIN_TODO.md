# 0. 관리자 로그인 / 접근

- **목표 동작**: `사이트 접속 → /admin 직접 입력 → (미인증) 관리자 로그인 화면 → 인증 성공 시 대시보드`. 로그인 화면은 자사몰 로그인과 동일한 디자인.
- **구현**:
  - 페이지 `app/admin/login/page.tsx` 신설 (자사몰 로그인 UI 재사용, 상단만 "관리자 로그인" 표기).
  - 미들웨어/가드: 어드민 진입 시 Supabase 세션 + `profiles.role='admin'`(또는 허용 이메일 화이트리스트) 확인 → 불충족이면 `/admin/login`으로 리다이렉트(현재의 "접근 권한 없음" 텍스트 대체).
  - 로그인 후 role 확인 실패 시 "관리자 권한이 없는 계정입니다" 안내 + 로그아웃.
- **연결**: 기존 로그인 로직(Supabase Auth) 재사용. role 컬럼이 없으면 `profiles.role` 추가(기본 'user').

# 1. 대시보드 (dashboard)

### 1-1. 상단 주문 단계 플로우 바 🆕 P1
- **현재**: 대시보드 상단이 자체 KPI 카드(오늘 주문/재주문/매출/환불률 등) 위주.
- **목표 동작**: 스마트스토어식 `신규주문 ▸ 배송준비 ▸ 배송중 ▸ 배송완료 ▸ 구매확정` 가로 플로우. 각 칸에 **실시간 건수**. 칸 클릭 → 주문관리가 그 상태로 필터된 채 열림. 우측 상단 "최근 HH:MM + 새로고침" 버튼.
- **구현**:
  - 상태 매핑: 신규주문=`paid`, 배송준비=`preparing`, 배송중=`shipped`, 배송완료=`delivered`, 구매확정=`confirmed`(없으면 신설 or delivered+N일 자동).
  - 카운트 쿼리: `orders`에서 status별 count(`supabase ... select('id',{count:'exact',head:true})`).
  - 칸 클릭 시 `router.push('/admin?panel=orders&status=preparing')` 형태 → 주문관리가 쿼리 읽어 필터 초기화.
  - 새로고침 = 카운트 재조회 + 시각 갱신.
- **연결**: 2-1 주문관리 단계바와 **같은 상태/필터 규칙 공유**(공통 상수 `ORDER_STAGES`).

### 1-2. 취소·반품·교환 / 판매지연 요약 카드 🆕 P1
- **목표**: 취소요청·반품요청·교환요청 / 발송지연·취소지연 등 건수 카드. 클릭 시 해당 목록.
- **구현**: 취소=환불요청 상태, 지연=기준시간 초과 미발송(예: paid 후 N일 미shipped) 계산식 정의. 카드 컴포넌트 재사용.
- **연결**: 19. 환불관리 지표와 동일 소스.

### 1-3. 판매 성과 그래프 🆕 P2
- **목표**: 일간/주간/월간 토글, 방문수·주문건수·구매전환율·객단가·결제금액 + 전기 대비 %.
- **구현**: 기존 `SalesChart` 확장. 방문수는 별도 집계 테이블 필요(없으면 추후/생략 표기). 토글 = 기존 `adm-seg-btn` 패턴.
- **연결**: 20. 정산 그래프와 데이터 소스 공유 가능.

### 1-4. 판매 TOP10 / 농가별 순위 🆕 P2
- **목표**: 많이 팔린 상품 TOP10, 농가별 판매 순위.
- **구현**: `order_items` 집계(product_id별 수량/매출 합) + product→farm 조인. 기간 필터 연동.
- **연결**: 4. 농가관리 "베스트 상품" 지표와 공유.

### 1-5. 기존 KPI 유지 위치 🔧 P3
- 기존 상단 지표(오늘 주문/재주문/매출/환불률, 방문자/신규회원/객단가/전환율)는 **2줄 보조 지표 섹션**으로 하단 유지. "재주문 고객수" 표기 방식만 개발자와 협의 후 확정.

# 2. 주문 관리 (orders)

### 2-1. 상단 단계 요약 바 🆕 P1
- **목표**: 대시보드 1-1과 동일한 단계바를 주문관리 상단에도. 클릭 = 그 상태 필터.
- **구현**: `ORDER_STAGES` 공통 상수 사용, 클릭 시 목록 필터 state 변경(페이지 이동 X, 즉시 필터).

### 2-2. 농가별 보기 + 발주서 다운로드 🆕 P1
- **현재**: 전체 주문 목록만.
- **목표 동작**: "전체 보기 ↔ 농가별 보기" 토글. 농가별 보기 시 농가 단위로 묶여 표시 → **농가별 발주서(엑셀/PDF) 다운로드** → 농가에 전달 → 농가가 보낸 송장 입력 → 발송처리. 운영 핵심 플로우.
- **구현**:
  - 주문 아이템에 농가 매핑(`order_items.product_id → products.farm_id`).
  - 농가별 그룹핑 뷰: 농가 선택 드롭다운 or 아코디언. 발주서 = 해당 농가의 미발송 주문아이템(상품/옵션/수량/수령인/주소/메모) 모음.
  - 다운로드: 엑셀(xlsx, 기존 엑셀다운 패턴 재사용) — 농가에 바로 전달 가능한 양식.
  - 송장 입력: 농가별 목록에서 행별 택배사/송장 입력 → "발송처리"(status `preparing→shipped`) + 배송시작 SMS(기존 `/api/notify`).
- **연결**: 4. 농가관리(farm 정보), 14/알림(송장 입력 시 SMS), 1-1 단계바 건수.

### 2-3. 조회 기준 + 기간 조회 🆕 P1
- **목표**: 기준 선택(결제일/결제완료일/주문일) + 기간(오늘/1주/1개월/3개월/직접입력) 후 검색.
- **구현**: 기준→정렬/필터 컬럼 매핑(`created_at`/`paid_at`), 기간 범위 쿼리(`gte/lte`). 네이버식 날짜 프리셋 버튼.

### 2-4. 페이지당 N개 + 페이지네이션 🆕 P2
- **목표**: 50/100개씩 셀렉트 + 페이지 이동.
- **구현**: 공통 페이징 컴포넌트(23-1)로. Supabase `range(from,to)`.

# 3. 상품 등록/관리 (products)

- **현재**: 카테고리 단일 선택.
- **목표**: 대분류 → 소분류 2단. 등록 시 대분류 선택하면 소분류 목록 갱신.
- **구현**:
  - `categories` 테이블에 `parent_id`(self FK) 추가 or `category_groups`+`categories`. 상품엔 소분류 id 저장(대분류는 소분류로 역참조).
  - 어드민 등록 폼: 대분류 select → 소분류 select(종속). 카테고리 관리 화면에서 대/소 추가.
- **연결**: 프론트 카테고리 페이지(모바일 개편 중 카테고리 2단과 **동일 스키마 공유**), 상품 필터.

### 3-2. 판매 상태 지표 카드 🆕 P2
- **목표**: 상단에 판매중/품절/판매중지/판매종료 건수 카드(클릭 시 필터).
- **구현**: `products.status`(또는 is_active+stock) 기준 count. 칸 클릭→목록 필터.

### 3-3. 상세페이지 등록 통합 🔧 P1
- **현재**: 상품 등록 후 별도 버튼으로 상세정보 작성.
- **목표**: 상품 등록 화면 안에서 상세페이지(에디터)까지 한 번에 작성·저장.
- **구현**: 등록 폼에 상세 에디터 섹션 인라인 포함, 저장 시 상품+상세 동시 insert. (에디터는 10-2 스마트스토어식과 동일 컴포넌트 공유)
- **연결**: 10. 이벤트, 11. 라운지 에디터와 공통 에디터.

### 3-4. 원산지 계층 입력 🆕 P2
- **목표**: 국내산 → 경기도 → 고양시 식 단계 입력.
- **구현**: 시도/시군구 데이터셋 기반 종속 select(국내산 선택 시 시도→시군구). 수입산은 국가 select. 저장은 `origin` 문자열 합성("국내산 경기도 고양시").

### 3-5. 가격/할인 영역 그룹화 🔧 P1
- **현재**: 정산가·할인율이 좌우로 흩어져 동선이 왔다갔다.
- **목표**: "판매금액" 관련 입력(판매가/할인/정산가)을 한 묶음 카드로 모아 배치.
- **구현**: 폼 레이아웃 재구성(섹션 카드). 로직 변경 없음, 배치만.

### 3-6. 할인율/할인액 양방향 🆕 P1
- **목표**: 할인을 율(%) 또는 액(원) 어느 쪽으로 입력해도 됨. **할인액으로 넣어도 소비자 화면엔 자동 %로 표기.**
- **구현**: 입력 토글(% / 원). 저장 시 `discount_type`('percent'|'fixed')+`discount_value` 보관. 소비자 표기용 % = `round(할인액/판매가*100)` 계산해 노출.
- **연결**: 상품 상세/카드의 할인% 뱃지 표시 로직.

### 3-7. 상위/하위 2단 옵션 🆕 P1
- **현재**: 옵션 그룹(group_name)+필수여부까지는 구현됨(가산형). 하지만 "상위 선택에 따라 하위가 바뀌는" 종속 구조는 아님.
- **목표**: 네이버식 종속 옵션 — 예: 방울토마토 → (1단계) 유기농/무농약 → (2단계) 1kg/2kg. 1단계 선택값에 따라 2단계 목록이 달라짐.
- **구현**:
  - 옵션 데이터에 `parent_option_id` 또는 조합(매트릭스) 구조. 등록 UI: 1단계 값 입력 → 각 값에 종속 2단계 값 입력(네이버 "옵션목록으로 적용"식 조합 테이블 생성).
  - 상품상세/장바구니: 1단계 select → 2단계 select(종속 갱신) → 합산가/재고는 조합 단위.
- **연결**: 상품상세 OptionDrawer, 장바구니 옵션 표기, 재고관리.

### 3-8. 단품 바로구매 🆕 P2 (Q1 결정: 옵션 없으면 바로구매)
- **목표**: 옵션이 없는 단품은 옵션 선택 없이 바로 구매. 옵션 있으면 기존처럼 선택.

### 3-9. 한 줄 설명 → 상품명에 묶기 🔧 P2

### 3-10. 당도(brix) 삭제 🔧 P1

# 4. 농가 관리 (farms)

### 4-1. 농가 유형(분류) 관리 + 필터 🆕 P2 (Q3 확정)
- **현재**: 자사배송/산지직송 2종으로 나뉨(고정).
- **목표**: 유형을 **추가 가능한 분류**로 만들어 향후 종류 확장. **유형별로 농가·상품을 필터/조회** 가능.
- **구현**:
  - `farm_types` 테이블(또는 settings의 목록): {id, name, color?}. 기본 2개(자사배송/산지직송) 시드.
  - 농가에 `farm_type_id`. 농가관리 상단에 유형 필터 탭(전체/자사배송/산지직송/…).
  - 어드민 설정에서 유형 추가/이름수정.
- **연결**: 3-13 상품등록의 자사/산지 우선 선택이 이 유형을 사용. 1-4 농가 순위/4-3 지표도 유형별 집계 가능.

# 7. 쿠폰 (coupon ▸ 쿠폰 관리 탭)

### 7-1. 상단 지표 🆕 P1
- 전체 쿠폰 / 활성 / 비활성 / 총 발급 / 총 사용량 카드.
- **구현**: coupons count by is_active, user_coupons 발급/사용 합계.

### 7-2. 발급 방식 기본값 = 다운로드 🔧 P1 (Q5)
- **현재**: 생성 시 공개/자동 여부가 헷갈림.
- **목표**: 기본을 **"다운로드형"**(회원이 받아야 보유). "자동 발급" 켜면 대상에게 자동 지급.
- **구현**: 생성 폼에 발급방식 라디오(다운로드/자동발급), `coupons.issue_type`. 다운로드형은 프론트 "쿠폰받기"에 노출, 자동발급형은 조건 충족 시 즉시 user_coupons insert.

### 7-3. 쿠폰 설명 필드 🆕 P2
- `coupons.description` 추가, 목록/상세 표기.

### 7-4. 목록 표기: 코드 노출 🔧 P2
- 목록에서 할인유형 대신 **쿠폰 코드** 우선 표기(멘트는 참고 이미지처럼 카드형으로 보기 좋게).

### 7-5. 개별(지목) 지급 숨김 🆕 P2 (Q6)
- **목표**: 특정 회원에게만 지급한 쿠폰은 **타 회원에게 안 보이게**.
- **구현**: `is_public=false`(지목 발급 전용) → 프론트 쿠폰 목록/다운로드에서 제외, 발급된 당사자 user_coupons에만 존재.

### 7-6. 회원가입 쿠폰팩 자동지급 + 생성기 🆕 P1 〔신규 요청〕
- **현재**: 회원가입 시 쿠폰팩(1만원) 지급이 제대로 설정 안 됨/방법 불명. 상품상세 비로그인 화면엔 **"5000원 쿠폰 받기"가 하드코딩**된 듯.
- **목표 동작**:
  1) 어드민에 **"회원가입 지급 쿠폰" 설정**(쿠폰팩 구성: 어떤 쿠폰들을 회원가입 시 줄지 + 금액).
  2) 회원가입 완료 시 그 쿠폰팩이 **자동으로 신규회원 계정에 지급**.
  3) 상품상세(비로그인)의 "○○원 쿠폰 받기" 문구의 **금액이 설정값과 연동**(하드코딩 제거) → 설정한 쿠폰팩 총액 표기.
- **구현**:
  - 쿠폰 생성기에 "회원가입 지급" 플래그(`signup_pack=true`) 또는 settings `signup_coupon_ids[]`.
  - 회원가입 후처리(Auth signup hook / signup API)에서 해당 쿠폰들을 user_coupons에 멱등 발급(친구초대 구조 참고).
  - 상품상세 배너 문구 = 설정된 회원가입 쿠폰팩 총액을 읽어 표기(하드코딩 "5000원" 제거).
- **연결**: 친구초대 쿠폰(자동지급, A7)과 동일한 멱등 발급 패턴 재사용. 회원가입 플로우, 상품상세 비로그인 배너.

### 7-7. 친구초대 쿠폰 자동발급 명시 🔧 P2 (A7)
- 이미 자동(첫 구매 배송완료 시 추천인에게 5,000원 쿠폰 멱등 발급). 화면에 "자동발급"으로 표기, 13-1 지급여부 표시와 연결.

- 환불요청/환불보류/진행중/환불완료/**환불불가** 건수 카드.
### 19-2. 기간+처리상태 필터 🆕 P1
- 조회기간 + 상태 필터(네이버식). (전체/고객신청/관리자취소 필터는 6/5 완료)
### 19-3. 환불 거부 + 사유 전달 🆕 P1
- **목표**: 환불 거부 상태 + 소비자에게 사유 전달.
- **구현**: refund_requests에 거부사유 필드, 거부 시 사유 입력 → 고객 마이페이지/문자로 전달.
- **연결**: 마이페이지 환불내역 표시, 14 문자.

# 20. 정산 관리 (settlement)

### 20-1. 기간 조회 + 일/월 그래프 🆕 P1

- **연결**: 1-3 판매성과, 2 주문 데이터.

# 21. 취향 프로파일 (tasteprofile)

### 21-1. 응답률 지표 🆕 P1
- 총 회원 대비 설문 응답률 등 명확한 카드.
### 21-2. 유형별 필터 🆕 P1
- 응답 유형별 보기 필터.
### 21-3. 응답 분포 분석 🆕 P1
- **목표**: 소비자가 고른 값(나이대 / 구매목적 등) 선택 **분포**를 집계해 마케팅에 활용.
- **구현**: 설문 응답 항목별 집계(막대/도넛). 기간/세그먼트 필터.
- **참고**: 데이터/로직은 변경 없이 **시각화/집계만**(취향진단 자체 로직 불변).

# Q&A 정리
- **Q1 단품 바로구매**: 옵션 0개면 바로구매, 있으면 옵션 선택 (3-8). → 확정.
- **Q2 맛 프로파일**: 클라이언트 추후 스펙 전달 → **보류**(3-15).
- **Q3 농가 유형**: 추가 가능한 분류 + 유형별 필터. 기본 자사배송/산지직송 시드 (4-1). → 확정.
- **Q4 리뷰 답변**: 판매자 답글 → 상세 리뷰 아래 노출 (5-2). → 확정.
- **Q5 쿠폰 발급 기본값**: 다운로드형 기본, 자동발급 옵션 (7-2). → 확정.
- **Q6 지목 지급 숨김**: is_public=false (7-5). → 확정.
- **Q8 입점문의**: 수동 발송, 메일/문자 템플릿 + 전화 번호표시 (15-2). → 확정.
- **A7 친구초대**: 이미 자동지급(첫 구매 배송완료 시 5,000원 쿠폰 멱등). 표기만 명시 (7-7,13).
- **신규 회원가입 쿠폰팩**: 자동지급 + 생성기 + 상품상세 금액 동적화 (7-6). → 진행.
